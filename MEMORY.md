## GIT-WORKFLOW

- `main` — продакшн. Деплоится на Railway автоматически. Трогать только когда правки проверены.
- `dev` — все правки идут сюда. После проверки — сливать в `main`.

**Порядок работы:**
1. Убедиться что находимся в `dev` (`git branch`)
2. Внести правки
3. Commit + push в `dev`
4. Проверить на продакшне (Railway деплоит `dev` — если настроено) или локально
5. Слить в `main`: `git checkout main && git merge dev && git push origin main && git checkout dev`

## АРХИТЕКТУРНЫЕ РЕШЕНИЯ — НЕ МЕНЯТЬ БЕЗ ПРИЧИНЫ

### 1. `enterApp(animate)` — единая точка входа в приложение
**Файл:** `public/index.html`, JS-секция
**Причина:** раньше было 3 разных места где показывался/скрывался интерфейс. Это приводило к багам: шапка появлялась на welcome-странице, упражнения были пустые, потому что `body.in-app` не добавлялся.
**Запрет:** не добавлять `display:none/block` или classList манипуляции с `#app` и `#welcome-page` в обход этой функции.

### 2. `body.in-app` — CSS-класс управляет видимостью шапки и приложения
**Файл:** `public/index.html`, CSS-секция
**Причина:** `.app-header { display: none }` переопределяется через `body.in-app .app-header { display: flex }`. Если убрать класс с body — шапка исчезнет или появится не там.
**Запрет:** не дублировать `display: flex` для `.app-header` без условия `body.in-app`.

### 3. Финальный шаг (#s6) — только через `openFinalStep()`
**Файл:** `public/index.html`, JS-секция
**Причина:** перед показом финала нужно рендерить ответы пользователя и делать API-запрос. Прямой `goStep(6)` покажет пустой экран.
**Запрет:** не вызывать `goStep(6)` напрямую.

### 4. `login_count` — счётчик заходов в БД
**Файл:** `server.js`, endpoint `/api/user/load`
**Причина:** добавлен через `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` — безопасно для существующей БД. Инкрементируется при каждом `/api/user/load`.
**Запрет:** не пересоздавать колонку через DROP — потеряются данные.

### 5. Ответы хранятся как `state.answers[moduleId][exId]`
**Файл:** `public/index.html`, JS-секция
**Структура:** `state.answers = { gaps: { ex1: '...', ex2: '...', ex3: '...' }, intent: {...}, ... }`
**Важно:** финальный шаг и `/api/final` собирают ответы именно из этой структуры. Если изменить формат — финал сломается.

## API ENDPOINTS

| Метод | URL | Что делает |

| POST | `/api/generate` | Генерирует упражнения для модуля через Anthropic |

## ИСТОРИЯ ВАЖНЫХ БАГОВ

### Баг: шапка показывалась на welcome-странице
**Причина:** второй CSS-блок `.app-header {}` имел `display: flex` и перебивал `display: none` из первого блока.
**Фикс:** `display: none` внутри основного блока, `body.in-app .app-header { display: flex }` отдельно после.

### Баг: упражнения были пустыми при восстановлении сессии
**Причина:** IIFE восстановления сессии добавлял `app.classList.add('visible')` но не добавлял `body.in-app` — CSS скрывал `#app`.
**Фикс:** рефакторинг в `enterApp(false)` который делает оба действия.

### Баг: логотип перекрывал контент на welcome-странице
**Причина:** логотип был `position: absolute; left: 50%; transform: translateX(-50%)`.
**Фикс:** обычный flex-элемент с `margin-bottom: 1.5rem`.

### Баг: график показывал только дни с данными
**Причина:** SQL возвращает только строки с регистрациями.
**Фикс:** функция `fillSeries()` на фронтенде генерирует полный диапазон дат и заполняет пропуски нулями.
